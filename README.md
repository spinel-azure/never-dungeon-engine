# Never Dungeon Engine (NDE)

Never Dungeon Engine（NDE）は、ブラウザ上で動作するクラシックスタイルの3DダンジョンRPGを制作するためのプロトタイプエンジンです。

現在は、10×10のセル境界壁式ダンジョンをランダム生成し、Canvas 2Dによる疑似3D視点で探索できる基盤を開発しています。

## 現在実装されている主な機能

### ダンジョン

- Canvas 2Dとレイキャストによる疑似3D描画
- 10×10セルのランダム迷路生成
- 初回と再生成時のランダムな開始地点、階層移動時の階段座標継承
- 全100セルへ到達できる迷路生成
- 行き止まりだけでなくループ経路も含む通路構成
- 床、セル境界壁、扉を分離したデータ管理
- 通常扉6枚、ボス部屋用の赤扉1枚、施錠用の黒扉1枚のランダム配置と開扉アニメーション
- 上り階段と下り階段の3D表示およびミニマップ表示
- 前進、後退、左右旋回のアニメーション
- 壁やマップ外への移動を防ぐ衝突判定

### 探索支援

- 探索済み範囲を記録するミニマップ
- 未探索セルのノイズ表示
- 壁、扉、階段、NPC、プレイヤー位置のマップ表示
- コンパス
- 階層／座標表示（オプションで切り替え可能）
- 探索済みセルだけを通ってスタート地点へ戻るオート帰還
- ランダム生成によるダンジョン再生成

### たいまつと気配ゲージ

- 正常に1マス移動するたびに減少するたいまつゲージ
- たいまつが尽きた場合の暗闇表示
- 0～100で蓄積する気配ゲージ
- 通常時は1歩ごとにランダムで+4～+8
- 暗闇では1歩ごとにランダムで+5～+10
- 壁への衝突、旋回、メニュー操作では気配が増加しない
- NPC・階段などの特殊マスではランダムエンカウント判定を行わない
- 気配100到達時の仮エンカウント通知
- 通知を閉じた後に気配ゲージをリセット

戦闘システムは未実装のため、現在は遭遇メッセージのみ表示します。

### NPCイベント

- NPCデータを `data/npcs.js` で一元管理
- NPCを進行不能が発生しない安全なセルへランダム配置
- ダンジョン内のNPCスプライトとミニマップ上の人型アイコン
- 正面1マスにNPCがいて壁や閉扉がない場合の気配メッセージ
- 気配最大時の遭遇オーバーレイとAボタンによる探索復帰
- ランダム生成開始からの探索時間を示すストップウォッチ
- 階層移動時のラップタイム表示と任意キー・タップによる解除
- 汎用オーバーレイイベントによるNPC会話表示
- Aボタンによる会話進行とBボタンによるキャンセル
- NPCとの遭遇回数を記録できるイベント管理
- 会話終了後のNPC消去と通路開放

現在実装されているNPCは「みかんにゃんこ」1体です。

### 宝箱イベント

- 赤・黒・金の宝箱を各1個、安全な床マスへランダム配置
- 階段・NPCとの重複、および配置による進行不能を回避
- 探索中は0度固定の透過画像をNPCと同じ遠近投影で表示
- 宝箱マスへ入ると黒オーバーレイ上にThree.js製の宝箱を表示
- Aボタンで2回転、開蓋、内部発光、光粒の開封演出
- Bボタンで開けずに直前のマスへ後退
- 現在の中身は仮実装のため空

### 階段イベント

- 上り階段で帰還確認を表示
- 下り階段で次の階層へ進む確認を表示
- Aボタンで未実装メッセージを表示
- Bボタンで確認を閉じ、その場にとどまる
- 階段上でキャンプメニューを開き、BACKまたはMAINで戻った場合の再確認

階層移動と帰還処理そのものは未実装です。

### UI・操作

- PC、タブレット、スマートフォン向けレスポンシブUI
- タッチ端末用バーチャルスティック
- タッチ端末用A／Bボタン
- キャンプメニューとオプション画面
- コンパスおよび階層／座標表示のON／OFF
- オプション画面からのランダム生成、オート帰還、たいまつ回復
- ゲーム用ローカルフォント `DotGothic16` と `k8x12`

## 操作方法

| 操作 | キーボード | 画面操作 |
| --- | --- | --- |
| 前進 | `↑` | 上ボタン／バーチャルスティック上 |
| 後退 | `↓` | 下ボタン／バーチャルスティック下 |
| 左旋回 | `←` | 左ボタン／バーチャルスティック左 |
| 右旋回 | `→` | 右ボタン／バーチャルスティック右 |
| 決定・会話を進める | `X` | Aボタン |
| キャンセル・キャンプメニュー | `Z` | Bボタン |

## 技術構成

- HTML
- CSS
- JavaScript
- ES Modules
- Canvas 2D API
- 外部ライブラリなし
- ビルド環境なし

公開本線は、`index.html`からES Modules版の`js/main.js`を`type="module"`で読み込みます。

## 開発ステータス

現在は探索エンジンのプロトタイプ段階です。

戦闘、階層移動、アイテム、キャラクター成長、シナリオ、BGM／SEなど、ゲーム本編に必要な機能は今後実装する予定です。キャンプメニュー内にも未実装項目が含まれています。

## GitHub Pagesでの公開について

NDEは静的ファイルのみで構成され、GitHub Pagesでの動作を本線としています。

ES Modulesを使用しているため、ローカルで確認する場合も`file://`による直開きではなく、HTTPサーバー経由で開いてください。
